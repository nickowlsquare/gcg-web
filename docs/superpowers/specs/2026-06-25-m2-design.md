# M2 lib/legality.ts — Legality Engine Design Spec

**Date:** 2026-06-25
**Milestone:** M2 — Legality Engine

---

## Goal

Build `lib/legality.ts` — the authoritative deck legality module used by the builder (M1), autofill engine (M3), and build page (M4). Replace the partial `validateDeck` in `lib/deck.ts` with a comprehensive, structured result.

---

## Return Type

```ts
export interface LegalityResult {
  isLegal: boolean     // true if and only if errors.length === 0
  errors: string[]     // hard rule violations — deck cannot be played
  warnings: string[]   // soft suggestions — deck is legal but suboptimal
}
```

---

## Hard Rules (errors)

These make `isLegal: false`:

| Rule | Error message |
|------|--------------|
| Main deck ≠ 50 cards | `"主牌組需要 50 張（目前 N 張）"` |
| Resource deck ≠ 10 cards | `"資源牌組需要 10 張（目前 N 張）"` |
| Any card in main deck with count > 4 | `"[card name] 超過 4 份上限（目前 N 份）"` |
| More than 2 colors across both decks | `"超過 2 種顏色（目前：Color1、Color2、Color3）"` |

Notes:
- Resource cards are exempt from the 4-copy limit (per game rules).
- Color check applies to the combined main + resource deck.
- The 4-copy check iterates all `mainDeck` entries; any with count > 4 add an error (addCard normally prevents this, but legality must still validate independently).

---

## Soft Rules (warnings)

These leave `isLegal: true` when the deck is otherwise legal:

| Rule | Warning message |
|------|----------------|
| Unit count outside 25–28 | `"Unit 建議 25–28 張（目前 N 張）"` |
| Pilot count outside 6–8 | `"Pilot 建議 6–8 張（目前 N 張）"` |
| Command count outside 8–10 | `"Command 建議 8–10 張（目前 N 張）"` |
| Base count outside 4–6 | `"Base 建議 4–6 張（目前 N 張）"` |

Warnings are computed from the main deck only (resource cards do not count toward type ratios).

---

## Function Signature

```ts
export function checkDeck(
  mainDeck: Record<string, number>,
  resourceDeck: Record<string, number>,
  allCards: Card[]
): LegalityResult
```

Pure function — no side effects, no mutation of inputs.

## DeckList Update

Add `warnings` prop alongside existing `errors`:

```ts
interface DeckListProps {
  // ... existing props
  errors: string[]
  warnings: string[]   // new
}
```

Warning display: same panel as errors but amber colour (`text-amber-400`, `border-amber-500/30`, `bg-amber-500/10`). Shown below errors when `warnings.length > 0`. Errors and warnings can both be visible simultaneously.

## Testing (lib/legality.test.ts)

All tests written before implementation (TDD):

**`checkDeck` — hard errors:**
- Main deck with exactly 50 cards → `isLegal: true`, no errors
- Main deck with 49 cards → error contains "主牌組"
- Main deck with 51 cards → error contains "主牌組"
- Resource deck with exactly 10 cards → no error
- Resource deck with 9 cards → error contains "資源"
- Resource deck with 11 cards → error contains "資源"
- Card with count 5 in main deck → error contains card name
- Card with count 4 → no 4-copy error
- 3 colors → error contains "顏色"
- 2 colors → no color error
- All four hard rules violated simultaneously → 4+ errors

**`checkDeck` — warnings:**
- Unit count 25 → no unit warning
- Unit count 24 → warning contains "Unit"
- Unit count 29 → warning contains "Unit"
- All four type counts in range → no warnings
- Legal deck (50/10) with bad ratios → `isLegal: true` but warnings present

**`checkDeck` — `isLegal`:**
- `isLegal: true` ↔ `errors.length === 0`
- `isLegal: false` when any error present

**Immutability:**
- `checkDeck` does not mutate its inputs
