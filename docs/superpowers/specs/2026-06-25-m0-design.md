# M0 設計文件 — GCG Web 專案骨架 + /cards 頁面

**日期：** 2026-06-25
**範圍：** 里程碑 M0 — 專案骨架、型別定義、樣本卡資料、/cards 瀏覽頁

---

## 1. 背景與目標

建立 GCG Web（Gundam Card Game 自動組牌網站）的地基。M0 完成後，開發者可以在瀏覽器看到一個有真實卡牌資料、可篩選的卡片瀏覽頁，確認技術棧與設計語言可行。

M0 不包含任何組牌或自動補牌邏輯，那些在 M1–M4 實現。

---

## 2. 技術棧

- **框架：** Next.js 14（App Router）
- **語言：** TypeScript（strict 模式）
- **樣式：** Tailwind CSS v3
- **測試：** Vitest（M0 暫無測試，M2 起必須覆蓋規則引擎）
- **部署：** Vercel

## 3. 視覺設計語言

### 3.1 主題方向
**Premium TCG**：深紫黑漸層底色、金色強調、玻璃感卡片。參考高質感 TCG 應用程式（遊戲王、寶可夢）。

### 3.2 顏色 Token
這些值擴展到 `tailwind.config.ts` 的 `theme.extend.colors`：

```ts
// tailwind.config.ts（參考值）
colors: {
  bg: {
    base:     "#0f0a1e",  // 最底層深紫黑
    surface:  "#1a1030",  // 卡片、面板背景
    elevated: "#221540",  // hover 狀態
  },
  accent: {
    gold:    "#f0c060",   // 標題、LR 徽章、重要標籤
  },
  game: {
    blue:   "#3b7dd8",
    green:  "#4a9f5c",
    red:    "#d44a4a",
    white:  "#c8c8d4",
    purple: "#9a60d0",
  },
}
```

### 3.3 卡型顏色語意（全站統一）
| 卡型 | 色碼 | 用途 |
|---|---|---|
| Unit | `#3b7dd8` | 藍色標籤、背景淡色 |
| Pilot | `#4a9f5c` | 綠色標籤 |
| Command | `#d47a2a` | 橙色標籤 |
| Base | `#7a4ab0` | 紫色標籤 |
| Resource | `#888888` | 灰色標籤 |

## 4. 資料模型

### 4.1 CardColor（新增 purple，vs 原計劃書）
```ts
export type CardColor = "blue" | "green" | "red" | "white" | "purple";
```

### 4.2 Card 型別（完整版，含 Pilot 專用欄位）
```ts
export interface Card {
  // 識別
  id: string;              // 卡號，如 "GD01-015"
  name: string;
  type: CardType;
  rarity: string;          // "C" | "U" | "R" | "LR" | "P"
  set: string;             // 如 "GD01"
  sourceTitle?: string;    // 原作系列名，如 "Mobile Suit Gundam SEED"

  // 遊戲屬性
  colors: CardColor[];     // 1+ 色；無色為 []
  level: number | null;
  cost: number | null;
  ap?: number;             // Unit / Base 攻擊力
  hp?: number;             // Unit / Base 生命值
  isLR?: boolean;          // 是否 LR

  // Pilot 專用（deckplanet API 揭示）
  apBoost?: number;        // Pilot 配上 Unit 後加的 AP
  hpBoost?: number;        // Pilot 配上 Unit 後加的 HP
  linkRequirement?: string; // Link 條件描述
  blockIcon?: number;      // block 圖標數量

  // 分類與效果
  traits: string[];        // 陣營/系列標籤，如 ["Earth Alliance"]
  keywords: string[];      // 關鍵字技能，如 ["Deploy", "Burst"]
  zones?: string[];        // 可出場區域，如 ["Space", "Earth"]
  text: string;            // 效果文字（純文字，無 HTML）

  // 合法性
  isBanned?: boolean;
  isLimited?: boolean;
}

export interface DeckEntry { id: string; count: number; }
export interface Deck {
  colors: CardColor[];     // ≤ 2
  main: DeckEntry[];       // 加總 = 50
  resource: DeckEntry[];   // 加總 = 10
}

export interface TopDeck {
  name: string;
  colors: CardColor[];
  keyCards: string[];
  strategy: Strategy;
  list?: DeckEntry[];
  source: string;
  date: string;
  placement?: number;
}
```

### 4.3 資料來源
- **樣本卡（M0）：** 從 `api.deckplanet.net/cardsearch/gundam_cards` 抓取 12 張真實卡牌，儲存為 `data/cards.json`。
- **長期策略（M1+）：** 評估是否持續依賴社群 API，或建立自己管理的資料庫。
- **版權：** 卡圖不在本 repo 儲存；如需顯示卡圖，連結至官方或社群圖源。

### 4.4 12 張樣本卡組成
| 卡型 | 數量 | 備註 |
|---|---|---|
| Unit | 5 | 含 2 張 isLR:true（如 Wing Zero、Barbatos） |
| Pilot | 3 | 含 linkRequirement 示例 |
| Command | 2 | 不同 keyword 示例 |
| Base | 2 | 不同顏色 |

## 5. 頁面設計

### 5.1 全局 Layout（`app/layout.tsx`）
- 深紫黑背景（`--bg-base`）
- 頂部 Navbar：Logo（金色 ✦ GCG）+ 導航連結（Cards / Builder / Counter）
- 全站 font：系統字型優先，monospace 用於卡號等技術性文字

### 5.2 /cards 頁面
**佈局：** 左側固定篩選欄 + 右側卡片網格

**篩選欄（`FilterSidebar.tsx`）：**
- 顏色篩選：5 個色塊按鈕（Blue/Green/Red/White/Purple），可多選
- 卡型篩選：Unit/Pilot/Command/Base 四個 toggle，可多選
- 兩組篩選即時聯動，無需按確認
- 手機版：篩選欄收摺，頂部顯示「篩選」按鈕，點擊展開 drawer

**卡片網格（`CardGrid.tsx` + `CardTile.tsx`）：**
- 響應式：桌面 4 欄、平板 3 欄、手機 2 欄
- 每張 `CardTile` 顯示：卡名、卡號、卡型色標、Level、Cost
  - Unit / Base：顯示 AP / HP
  - Pilot：顯示 AP Boost / HP Boost
  - LR 卡：金色邊框 + ⭐ LR 徽章

**篩選邏輯（`lib/cards.ts`）：**
```ts
// 純函式，不依賴 React state
export function filterCards(
  cards: Card[],
  colors: CardColor[],
  types: CardType[]
): Card[]
```
- `colors` 空陣列 = 顯示所有顏色
- `types` 空陣列 = 顯示所有類型
- 顏色篩選：`card.colors` 與所選顏色有交集即顯示（OR 邏輯）
- 卡型篩選：完全匹配
