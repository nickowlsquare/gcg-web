# M1 /builder — Manual Deck Builder Design Spec

**Date:** 2026-06-25
**Milestone:** M1 — Manual Deck Builder

---

## Goal

Build a `/builder` page where users can browse all cards, add/remove them to a deck, and see live statistics (card type breakdown + cost curve). No saving or exporting in this milestone.

---

## Layout

Two-panel split:
- **Left panel (2/3 width):** Card browser — search bar + color/type filter toggles + card image grid
- **Right panel (1/3 width):** Deck panel — stats section on top (type counts + cost curve), scrollable deck list below

On mobile: panels stack vertically (left on top, right below).

---

## Left Panel — Card Browser

### Filters
Identical filter controls to `/cards` page:
- Text search input (filters by card name, case-insensitive)
- 5 color toggle buttons: Blue, Green, Red, White, Purple
- 4 card type toggle buttons: Unit, Pilot, Command, Base (Resource cards are not browsable — added separately)
- Filtered count display

### Card Grid
- Responsive grid: 2 cols mobile → 3 sm → 4 lg
- Uses existing `CardTile` component, extended with:
  - **Hover overlay:** Semi-transparent dark overlay appears on hover, showing a `+` button in the centre
  - **Count badge:** If the card is already in the deck, a small badge (e.g. `×2`) appears in the top-left corner of the tile
  - Clicking `+` button calls `addCard` (enforces all limits)

---

## Right Panel — Deck Panel

### Stats Section (pinned to top)
Two sub-sections side-by-side:

**Card Type Counts:**
- Shows count for each type: Unit, Pilot, Command, Base
- Each count shown in its type colour (matching Tailwind tokens: `cardtype-unit`, `cardtype-pilot`, `cardtype-command`, `cardtype-base`)
- Shows recommended range beneath each count (e.g. Unit: 25–28)

**Cost Curve:**
- Horizontal bar chart, bars aligned to bottom
- X-axis = cost value (1–6+), Y-axis = number of cards at that cost
- Bars rendered as simple `div` elements with proportional heights

### Deck List (scrollable)
Scrollable list below the stats:

**Main Deck section** (label: `MAIN DECK (n/50)`)
- One row per unique card
- Row layout: `[Card name] [Type badge] [− | n | +]`
- Card name colour matches rarity (LR = gold, else white/grey)
- Clicking `−` calls `removeCard` (removes at 0 — row disappears)
- Clicking `+` calls `addCard` (respects max 4 + max 50 limits)

**Resource Deck section** (label: `RESOURCE (n/10)`)
- Same row format as main deck
- Separate list below main deck in the same scrollable area

### Validation Errors
Inline error/warning bar below the deck list (only visible when there are issues):
- Over 50 main cards → error
- Over 10 resource cards → error
- Over 4 copies of a card → error (addCard prevents this; shown as disabled `+`)
- More than 2 colours in deck → warning (non-blocking; user can still add cards)

## Pure Functions — `lib/deck.ts` (all new)

### `addCard`
```ts
function addCard(
  deck: Record<string, number>,
  card: Card,
  isResource: boolean
): Record<string, number>
```
Rules enforced:
- Max 4 copies of any single card
- Max 50 cards in main deck (if `!isResource`)
- Max 10 cards in resource deck (if `isResource`)
- Returns unchanged deck if any limit is hit

### `removeCard`
```ts
function removeCard(
  deck: Record<string, number>,
  cardId: string
): Record<string, number>
```
- Decrements count by 1
- Removes key entirely when count reaches 0
- Returns unchanged deck if card not present

### `getDeckStats`
```ts
function getDeckStats(
  deck: Record<string, number>,
  allCards: Card[]
): {
  typeCounts: Record<CardType, number>
  costCurve: Record<number, number>   // cost → total copies at that cost
}
```

### `getDeckColors`
```ts
function getDeckColors(
  deck: Record<string, number>,
  allCards: Card[]
): CardColor[]
```
Returns deduplicated list of all colors present in the deck (for the 2-color validation check).

### `validateDeck`
```ts
function validateDeck(
  mainDeck: Record<string, number>,
  resourceDeck: Record<string, number>,
  allCards: Card[]
): string[]
```
Returns array of human-readable error/warning strings:
- `"主牌組超過 50 張（目前 n 張）"`
- `"資源牌組超過 10 張（目前 n 張）"`
- `"超過 2 種顏色（目前：Blue, Green, Red）"`

New optional props:
```ts
interface CardTileProps {
  card: Card

}
```

New optional props:

## What M1 Does NOT Include

- Saving or loading decks (localStorage, database, URL params)
- Exporting to any format
- Auto-fill / recommendation engine
- Win rate or strategy suggestions
- Drag-and-drop reordering
- Resource card browsing in left panel. Our current `cards.json` contains no `resource`-type cards, so the Resource Deck section in the right panel will render as an empty placeholder (label visible, no rows). The section is built and wired up so it works automatically once resource card data is added in a future milestone.

---

## Testing

All pure functions in `lib/deck.ts` must have Vitest unit tests before implementation (`lib/deck.test.ts`):
- `addCard`: enforce 4-copy limit, 50-card main limit, 10-card resource limit, happy path
- `removeCard`: decrement, remove at zero, no-op if absent
- `getDeckStats`: correct type counts, correct cost curve aggregation
- `getDeckColors`: deduplication, correct color extraction
- `validateDeck`: error messages for each violation, empty array when valid

Component tests are not required for M1.
