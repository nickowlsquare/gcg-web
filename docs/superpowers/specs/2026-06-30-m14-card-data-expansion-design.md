# M14 — Card Data Expansion Design

## Goal

Expand `data/cards.json` from 48 cards to cover all cards in **GD01–GD04**, **ST01–ST10**, and **EB01**, while upgrading the TypeScript schema to support the new card types and colours introduced in those sets.

## Architecture

Schema changes come first (types → UI → validation test), then card data is added set by set, each set committed independently. No new pages or routes. Existing 48 cards are preserved unchanged. All filtering, search, and drawer logic already works generically and requires no changes beyond the type-guard additions for new types.

**Tech Stack:** Next.js 14 App Router, TypeScript strict, Tailwind CSS v3, Vitest. No new dependencies.

---

## Scope Notes

- **Sets in scope:** GD01, GD02, GD03, GD04, EB01, ST01–ST10
- **Sets out of scope:** Edition Beta, Promo cards, Basic Cards, Other Product Cards
- **`topdecks.json` unchanged:** top-deck list stays at 20 decks
- **Data source:** Claude manually reads `gundam-gcg.com/en/cards/` per set — no automated scraping
- **Copyright constraint:** card text transcribed for app functionality only; no bulk redistribution

## Section 1 — Schema Changes

### `types/card.ts`

```ts
export type CardColor = 'blue' | 'green' | 'red' | 'purple' | 'white'

export type CardType =
  | 'unit' | 'pilot' | 'command' | 'base' | 'resource'
  | 'ex_base' | 'ex_resource' | 'unit_token'
```

### New type stat groupings

| Type | Stats shown |
|------|------------|
| `unit`, `unit_token` | AP, HP, Cost, Level |
| `base`, `ex_base` | AP, HP, Cost, Level |
| `pilot` | AP Boost, HP Boost, Cost, Level |
| `command`, `resource`, `ex_resource` | Cost, Level |

`unit_token` entries use `cost: 0` in JSON (tokens are generated by card effects, not played from hand).

### `data/cards.json`

No structural change — same JSON shape. New cards follow identical format to existing 48.

### `lib/cards.ts`

`filterCards` and `searchCards` already use `card.type === t` and string includes — no changes needed. New types auto-supported.

### `lib/legality.ts`

Deck rules (max 3 copies, 50-card limit) apply to all types unchanged.

## Section 2 — UI Updates

### `components/FilterSidebar.tsx`

**Colours array** — extend from `['blue', 'green']` to:
```ts
const COLORS: CardColor[] = ['blue', 'green', 'red', 'purple', 'white']
```

Tailwind classes per new colour:
- `red` → `border-red-500 text-red-400 bg-red-500/20`
- `purple` → `border-purple-500 text-purple-400 bg-purple-500/20`
- `white` → `border-slate-300 text-slate-300 bg-slate-300/20`

**Types array** — add:
```ts
{ value: 'ex_base',     label: 'EX Base'   }
{ value: 'ex_resource', label: 'EX Resource'}
{ value: 'unit_token',  label: 'Token'      }
```

### `components/CardTile.tsx`

`CARD_TYPE_COLORS` map — add entries:
```ts
ex_base:    'border-emerald-400 text-emerald-300',  // same family as base
ex_resource:'border-gray-400 text-gray-300',         // same family as resource
unit_token: 'border-orange-400 text-orange-300',     // distinct orange for tokens
```

### `components/CardDetailDrawer.tsx`

Stats section type guards — update to:
```ts
const isUnitLike = ['unit', 'unit_token'].includes(card.type)
const isBaseLike  = ['base', 'ex_base'].includes(card.type)
const isPilot     = card.type === 'pilot'
// everything else (command, resource, ex_resource) → Cost + Level only
```

## Section 3 — Data Entry Strategy

### Priority order

| Phase | Set | Cards currently | Target | ~New |
|-------|-----|----------------|--------|------|
| 1 | GD01 Newtype Rising | 13 | ~60 | ~47 |
| 2 | GD02 Dual Impact | 6 | ~60 | ~54 |
| 3 | GD03 Steel Requiem | 5 | ~60 | ~55 |
| 4 | GD04 Phantom Aria | 0 | ~60 | ~60 |
| 5 | EB01 Eternal Nexus | 0 | ~30 | ~30 |
| 6 | ST01–ST10 | 24 | ~200 | ~176 |

### Per-set workflow

1. Browse `gundam-gcg.com/en/cards/` filtered by set
2. For each card, record: `id`, `name`, `type`, `color`, `rarity`, `ap`, `hp`, `apBoost`, `hpBoost`, `cost`, `level`, `traits[]`, `keywords[]`, `text`, `linkRequirement?`, `isLR?`
3. Append to `data/cards.json` (existing cards unchanged)
4. Commit after each set

### Validation

```ts
  }
})
```

Existing topdecks ID reference test continues to run unchanged.

## Error Handling

- Cards with `unit_token` type and missing AP/HP → display `—` (same fallback as existing cards with null stats)
- New colours with no matching Tailwind class → FilterSidebar falls back to `border-white/20 text-white/50` (existing default)
- Any card added with unknown type → `CardTile` uses existing `resource` fallback colour

## Testing

- `lib/cards.test.ts`: 1 new required-fields test
- UI changes: no new unit tests — rendering verified manually after each set phase
- Each phase ends with `npm run build` to confirm TypeScript strict mode passes
